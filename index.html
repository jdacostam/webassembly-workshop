// ¿por qué a veces JS y Wasm son similares?

JavaScript moderno tiene un motor avanzado que optimiza el código en tiempo de ejecución. Para operaciones simples como suma de enteros, puede ser tan eficiente que WebAssembly no muestra una ventaja significativa.

Dónde sí gana WebAssembly:
· Cómputo complejo con tipos estrictos (sin boxing de números)
· Código portado desde C/C++/Rust ya optimizado
· Arranque predecible: Wasm no tiene fase de "calentamiento" JIT
· Operaciones SIMD (múltiples datos en paralelo)
· Algoritmos que el JIT no logra inferir los tipos

Regla práctica: usa JS para lógica de UI y APIs REST. Usa Wasm cuando tengas código C/Rust existente o necesites rendimiento predecible y portátil.
